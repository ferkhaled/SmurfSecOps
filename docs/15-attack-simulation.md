# 15 - Attack Simulation

Attack simulation is included to teach safe, observable security exercises.
The goal is not chaos for its own sake.
The goal is to practice detection, response, and hardening.

## Actor

- `gargamel` plays the attacker in the tutorial storyline

## Safety rules

- only run simulations in your lab
- never point tooling at systems you do not own or control
- document what you changed before each exercise
- clean up and reset after each scenario

## Starter scenarios

- container shell access
- suspicious network activity
- vulnerable image usage

See `attacks/README.md` and the scenario files for the walkthroughs.
